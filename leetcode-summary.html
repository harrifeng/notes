1 第二章 贪心算法

1.1 算法解释

  • 所谓贪心算法或者贪心策略,是指保证每次操作都是局部最优,从而得到的结果是全局最优

1.2 分配问题

1.2.1 455 Assign Cookies (Easy)

  1. 题目描述
    • 有一群孩子和一堆饼干，每个孩子有一个饥饿度，每个饼干都有一个大小。每个孩子只能吃 一个饼干，且只有饼干的大小不小于孩子的饥饿度时，这个孩子才能吃饱。求解最多有多少孩子 可以吃饱。
  2. 输入输出样例

    • 输入两个数组，分别代表孩子的饥饿度和饼干的大小。输出最多有多少孩子可以吃饱的数量。 

      Input: [1,2], [1,2,3]
Output: 2
    • 在这个样例中，我们可以给两个孩子喂 [1,2]、[1,3]、[2,3] 这三种组合的任意一种。
  3. 题解
    • 饥饿度最小的孩子最容易吃饱,所以先满足容易满足的孩子,毕竟我们的kpi是吃饱孩子的个数,一个容易吃饱和一个不容易吃饱的孩子对最终的贡献都是1
    • 我们需要先满足最容易吃饱的孩子,而且还要用能满足孩子的最小的饼干(贪心)
    • 满足第一个孩子之后,后续采取相同的策略,直到满足条件的饼干不存在

    • 总结一下这里的贪心策略 

      给剩余孩子里最小饥饿度的孩子分配最小的能饱腹的饼干
    • 具体实现是: 把孩子和饼干分别排序,然后从"饥饿度最小的孩子"和"大小最小的饼干"出发来进行遍历,直到饼干或者孩子遍历完

    • 代码如下: 

      from typing import List


class Solution:
    def findContentChildren(self, g: List[int], s: List[int]) -> int:
        needs = sorted(g)
        cookies = sorted(s)
        beg_n = 0
        beg_c = 0

        while beg_n < len(needs) and beg_c < len(cookies):
            if cookies[beg_c] >= needs[beg_n]:
                beg_n += 1
            beg_c += 1
        return beg_n

1.2.2 135 Candy (Hard)

  1. 题目描述
    • 一群孩子站成一排，每一个孩子有自己的评分。现在需要给这些孩子发糖果，规则是如果一 个孩子的评分 比自己身旁的一个孩子要高，那么这个孩子就必须得到比身旁孩子更多的糖果;所 有孩子至少要有一个糖果。 求解最少需要多少个糖果。
  2. 输入输出样例

    • 输入是一个数组，表示孩子的评分。输出是最少糖果的数量。 

      Input: [1,0,2]
Output: 5
Explain: 你可以分别给这三个孩子分发 2、1、2 颗糖果

Input:[1,2,2]
OUtput:4
Explain:你可以分别给这三个孩子分发 1、2、1 颗糖果。第三个孩子只得到 1 颗糖果，这已满足上述两个条件。
  3. 题解
    • 这道题目的难点在于要和两侧进行比较,如果遍历的时候需要考虑两边,一般都不好设计
    • 一般需要两侧比较的,都要改成单侧比较,然后从左到右,再从右到左,做两遍
    • 具体实现方法如下:
      • 首先所有孩子的糖果初始化为1
      • 第一遍从左到右遍历,如果右边孩子的评分比左边高,则右边孩子的糖果数目更新为左边孩子的数目+1
      • 第二遍从右到左遍历,如果左边孩子的评分比右边高(且左边孩子的糖果数目小于等于右边),则左边孩子的 糖果数目更新为右边孩子的数目+1

    • 代码如下 

      from typing import List


class Solution:
    def candy(self, ratings: List[int]) -> int:
        ret = [1] * len(ratings)
        for i in range(1, len(ratings)):
            if ratings[i] > ratings[i - 1]:
                if ret[i] <= ret[i - 1]:
                    ret[i] = ret[i - 1] + 1
            elif ratings[i] < ratings[i - 1]:
                if ret[i] >= ret[i - 1]:
                    ret[i - 1] = ret[i] + 1
        for i in range(len(ratings) - 1)[::-1]:
            if ratings[i] > ratings[i + 1]:
                if ret[i] <= ret[i + 1]:
                    ret[i] = ret[i + 1] + 1
            elif ratings[i] < ratings[i + 1]:
                if ret[i] >= ret[i + 1]:
                    ret[i + 1] = ret[i] + 1
        return sum(ret)

1.3 区间问题

1.3.1 435 Non-overlapping Intervals (Medium)

  1. 题目描述
    • 给定多个区间，计算让这些区间互不重叠所需要移除区间的最少个数。起止相连不算重叠
  2. 输入输出样例

    • 输入是一个数组，数组由多个长度固定为 2 的数组组成，表示区间的开始和结尾。输出一个 整数，表示需要移除的区间数量。 

      Input: [[1,2], [2,4], [1,3]]
Output: 1
    • 在这个样例中，我们可以移除区间 [1,3]，使得剩余的区间 [[1,2], [2,4]] 互不重叠。
  3. 题解
    • 求最少的移除的区间的个数,等同于尽量多保留不重叠的区间

    • 然后在保留区间的时候,区间的结尾比开头重要,这是这道题的题眼.选择区间结尾越小,留给其他区间的空间 就越大,就越能保留更多的区间. y因此我们采取的贪心策略为 

      优先保留结尾小且不相交的区间
    • 具体实现方法为:
      • 先把区间按照结尾的大小进行排序
      • 每次选择结尾最小,而且和前一个区间不重叠的区间

    • 代码如下 

      from typing import List


class Solution:
    def eraseOverlapIntervals(self, intervals: List[List[int]]) -> int:
        intervals = sorted(intervals, key=lambda x: x[1])
        N = len(intervals)
        removed = 0
        prev = intervals[0][1]
        for i in range(1, N):
            if intervals[i][0] < prev:
                removed += 1
            else:
                prev = intervals[i][1]
        return removed

2 第三章 玩转双指针

2.1 算法解释

  • 双指针主要用于遍历数组,两个指针指向不同的元素,从而协同完成任务
  • 如果两个指针指向同一个数组,遍历方向相同,但是不会相交,这种情况被称之为滑动窗口,在区间搜索中会经常用到
  • 如果两个指针指向同一个数组,遍历方向相反,那么往往是用在搜索排序好的数组

2.2 Two Sum问题

2.2.1 167 Two Sum II - input array is sorted (Easy)

  1. 题目描述
    • 在一个增序的整数数组里找到两个数，使它们的和为给定值。已知有且只有一对解。
  2. 输入输出样例

    • 输入是一个数组(numbers)和一个给定值(target)。输出是两个数的位置，从 1 开始计数 

      Input: numbers = [2,7,11,15], target = 9
Output: [1,2]
  3. 题解
    • 数组排序好,且需要遍历,那么双指针就是比较好的办法
    • 设置第一个位置是beg,最后一个位置是end
    • 两个指针一块朝中间赶,赶的策略是:
      • 如果beg和end的和等于target,那么找到结果返回
      • 如果beg和end的和小于target,那么beg增大,因为beg增大,beg和end的和增大
      • 如果beg和end的和大于target,那么end减小,因为end减小,beg和end的和减小

    • 代码如下: 

      class Solution:
    def twoSum(self, numbers: List[int], target: int) -> List[int]:
        if len(numbers) < 2:
            return []
        beg = 0
        end = len(numbers) - 1
        while beg < end:
            if numbers[beg] + numbers[end] == target:
                return [beg + 1, end + 1]
            elif numbers[beg] + numbers[end] < target:
                beg += 1
            else:
                end -= 1
        return []

2.3 归并两个有序数组

2.3.1 88 Merge Sorted Array (Easy)

  1. 题目描述
    • 给定两个有序数组，把两个数组合并为一个
  2. 输入输出样例

    • 输入是两个数组,把合并好的信息存放到第一个数组 

      Input: nums1 = [1,2,3,0,0,0], m = 3, nums2 = [2,5,6], n = 3
Output: nums1 = [1,2,2,3,5,6]
  3. 题解
    • 排序数组,还是两个,这两个信息告诉我们,大概率是一个two pointer问题
    • 我们把两个指针指向两个数组的末尾:
      • 一个是m - 1
      • 一个是n - 1
    • 我们还需要第三个指针来指向要更新的位置m+n-1
    • 更新的退出条件是:
      • m最后大于0,那么就直接结束了,因为m个剩余数字已经在正确的位置上了
      • n最后大于0,那么要把nums2剩下的n个数移动到nums1的前面

    • 代码如下 

      from typing import List


class Solution:
    def merge(
        self, nums1: List[int], m: int, nums2: List[int], n: int
    ) -> None:
        """
        Do not return anything, modify nums1 in-place instead.
        """
        pos = m + n - 1
        m -= 1
        n -= 1

        while m >= 0 and n >= 0:
            if nums1[m] > nums2[n]:
                nums1[pos] = nums1[m]
                m -= 1
            else:
                nums1[pos] = nums2[n]
                n -= 1
            pos -= 1
        if n >= 0:
            nums1[: n + 1] = nums2[: n + 1]
        return

2.4 快慢指针

2.4.1 142. Linked List Cycle II (Medium)

  1. 题目描述
    • 给定一个链表，如果有环路，找出环路的开始点
  2. 输入输出样例
    • 输入是一个链表，输出是链表的一个节点。如果没有环路，返回一个空指针。
  3. 题解
    • 这是一道链表找环路的问题,有一个通用的解法-快慢指针法(Floyed判圈法):
      • 给定两个指针slow每次走一步,fast每次走两步
      • 如果fast可以走到尽头,说明没有环路
      • 如果fast可以无限走下去,那么说明一定有环路,而且在某个时刻slow和fast会相遇
    • 除了判圈,我们还可以寻找环开始的地方:
      • 如果存在环
      • 当slow和fast第一次相遇时,将fast重新移动到链表开头,slow保持在原位置
      • slow和fast同事都前进一步,当slow和fast下一次相遇时,相遇的点就是环路开始的点
    • 针对寻找环的开始位置,有如下证明:

      • 图142.png

        Figure 1: leetcode/142.png

      • 假设slow走过的节点数目为 \(N_{slow}\) , AB = n, SA = m, 环的长度为L,a为slow绕过的圈数,那么有

        \begin{equation} N_{slow} = m + n + a \cdot L \end{equation}

      • 假设fast绕过的圈数为b,而且fast第一次相遇的时候,走过的节点数是slow的两倍,那么有

        \begin{equation} 2 \cdot N_{slow} = m + n + b \cdot L \end{equation}

      • 左右上面的式子,左边减去 \(N_{slow}\), 右边减去 \(m + n + a \cdot L\),得到

        \begin{equation} N_{slow} = (b - a) \cdot L \end{equation}

      • 最终我们得到了 \(N_{slow} = (b - a) \cdot L\) 也就是说slow在第一次相遇时候,走过的节点数是L的整数倍,也就是说

        \begin{equation} m + n = L \end{equation}
    • \(m + n=L\) 也就意味着m和L-n相等, slow从起点到环开始点走了m, fast从相遇点到环开始点也走了m,也就是L-n

    • 代码如下 

      from util.list_node import ListNode


class Solution:
    def detectCycle(self, head: ListNode) -> ListNode:
        if head is None or head.next is None:
            return None
        fast = head
        slow = head
        while True:
            if fast.next is None or fast.next.next is None:
                return None
            fast = fast.next.next
            slow = slow.next
            if fast == slow:
                break
        fast = head
        while fast != slow:
            fast = fast.next
            slow = slow.next
        return fast

2.5 滑动窗口

2.5.1 76. Minimum Window Substring (Hard)

  1. 题目描述
    • 给定两个字符串 S 和 T ，求 S 中包含 T 所有字符的最短连续子字符串的长度，同时要求时间复杂度不得超过 O(n)。
  2. 输入输出样例

    • 输入是两个字符串 S 和 T，输出是一个 S 字符串的子串。 

      Input: S = "ADOBECODEBANC", T = "ABC"
Output: "BANC"
    • 在这个样例中，S 中同时包含一个 A、一个 B、一个 C 的最短子字符串是“BANC”。
  3. 题解
    • 所谓滑动窗口问题,一般有如下的特征:
      • 求极值(最大或者最小值)
      • 两个指针运动方向一致,两个指针之间的部分叫做窗口
      • 一般数据不是排序的
    • 本题总体思路是双指针,但是还有其他难点:
      1. 双指针谁先移动
      2. 如何处理T中的重复,比如T是"AAB",有两个A
      3. 如何处理S中的重复,比如S="AAABC", T="ABC"的情况下,如何跳过前两个A
      4. 因为要返回字符串,如何存储区间
    • 下面算这几个难点的解决方案:
      1. 其实谁先移动都可以,但是我们选择让back先移动,因为front的移动范围就是当前位置,到小于等于back(注意这个等于)
      2. 对于T中的重复,我们使用字典来记录summary[A] = 2
      3. 对于S中的重复,我们需要使用一个cnt来记录"真实需要的数目",比如我们只需要3个(ABC),那么AAABC前 两个AA移动的之后,cnt的值并不改动(可以通过summary[A]的大小来判断是否真的改动cnt)

    • 代码如下 

      class Solution:
    def minWindow(self, s: str, t: str) -> str:
        cnt = len(t)
        from collections import defaultdict

        summary = defaultdict(int)
        for one in t:
            summary[one] += 1

        back = 0
        front = 0

        minv = (0, float("inf"))

        for back, curr in enumerate(s):
            if curr in summary:
                summary[curr] -= 1
                if summary[curr] >= 0:
                    cnt -= 1
                    while cnt == 0 and front <= back:
                        if s[front] in summary:
                            summary[s[front]] += 1
                            if summary[s[front]] > 0:
                                cnt += 1
                                if back - front < minv[1] - minv[0]:
                                    minv = (front, back)
                        front += 1

        if minv[1] > len(s):
            return ""
        return s[minv[0] : minv[1] + 1]
